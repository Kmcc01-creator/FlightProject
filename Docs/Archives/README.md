# Archives

This directory is the holding area for older documentation that is still worth preserving but should no longer compete with the current architecture and workflow docs.

Use this folder for:

- superseded architectural paradigms
- retired implementation plans
- older project pitches or conceptual frames
- preserved migration snapshots that are no longer current

## Status

Treat this folder as `Historical / Archived`.

Nothing here should be assumed active unless a current document links back to it explicitly for historical context.

## What Belongs Here

Good archive candidates:

- documents replaced by newer canonical docs
- one-time review prompts after the problem is resolved
- design notes that describe an older project identity
- plans kept only for historical reasoning

Bad archive candidates:

- current architecture references
- active workflow plans
- live build/test status
- practical environment setup docs

## Recommended Archival Pattern

When moving a file here:

1. add a short archival status banner at the top if it does not already have one
2. point back to the current canonical doc that replaced it
3. prefer preserving the original filename unless there is a strong reason to rename it

## Relationship To Other Folders

- `Docs/Architecture`
  current subsystem and runtime-model direction
- `Docs/Workflow`
  active implementation plans and current status
- `Docs/DevNotes`
  dated engineering notes that may still have reference value
- `Docs/Reviews`
  review prompts/supporting material, some of which may later move here when fully archival

If a document is no longer current and no longer useful as a live reference, `Archives` is the right destination.
