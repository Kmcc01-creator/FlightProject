# Reviews

This directory contains review prompts and review-oriented reference material.

Use these when you need an externalized review frame for a subsystem or implementation slice.

## Status

Treat this folder as `Supporting Material`.

Review docs can be valuable for framing audits and external analysis, but they are not canonical sources of truth on their own.

## Contents

- [ReflectionSystemReview_Prompt.md](ReflectionSystemReview_Prompt.md)

## Guidance

- keep prompts short and clearly dated
- point review prompts back to canonical architecture/workflow docs
- when a prompt reflects a resolved problem, mark it as archival rather than silently leaving it “live”

If this directory grows, split into:

- `Current review prompts`
- `Archived review prompts`
